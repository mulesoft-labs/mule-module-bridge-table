Class Overview

This connector offers the functionality of a bridge table (a.k.a. junction table, cross-reference table, join table, map table, intersection table, linking table, many-to-many resolver, link table, pairing table, pivot table, look-up table, or association table) to offer mapping of two values (keys)

Basically it supports inserting two keys (key1 and key2) and perform direct and reverse lookups:

  • Direct Lookup: Providing key1 you can retrieve the value of key2
  • Reverse Lookup: Providing key2 you can retrieve the value of key1

The relationship cardinality between key1 and key2 is 1 to 1. This means that there can only be one value of key2 linked to a value of key1 (and vicecersa).

The module exposes opertations to insert, update, delete and list keys too.

private static final Logger LOGGER

private boolean allowEmptyKeys

Whether empty keys are allowed. An empty key is null or the empty string ("").

private boolean autoCreateTable

If the bridge table doesn't exist it will be created automatically if this flag is set to true.

private BridgeTable bridgeTable

private DataSource dataSource

When the type is DATA_SOURCE this is the reference to the DataSource object that should be pre-configured before it can be used in this module. The name of this attribute is dataSource-ref

private String key1Name

Descriptive name for key1. This value may be used as a database table column name, so it needs to be a valid as so.

private KeyType key1Type

Data type of key1. This will be used to create the correct structure type. For example, if the implementation is done with a database then this attribute will be used to define the correct column type (STRING => varchar)

private String key2Name

Descriptive name for key2. This value may be used as a database table column name, so it needs to be a valid as so.

private KeyType key2Type

Data type of key2. This will be used to create the correct structure type. For example, if the implementation is done with a database then this attribute will be used to define the correct column type (STRING => varchar)

private boolean returnValueAsString

Whether all values should be returned as strings or not.

private String tableName

Name of the bridge table. For example if bridge table is implemented in a Database, this will be the name of the relational table

private Type type

Type or concreate implementation of the bridge table. Supported types are:

  • DATA_SOURCE: Implementation that receives a Data Source object and uses Database tables through JDBC
